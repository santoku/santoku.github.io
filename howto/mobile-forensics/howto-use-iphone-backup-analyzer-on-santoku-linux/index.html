HOWTO use iPhone Backup Analyzer on Santoku Linux


This HOWTO will guide you through a simple overview on how to use iPhone Backup Analyzer (iPBA) and get the most out of your iOS backup analysis on Santoku Linux.

Contents

  • What you will need
  • Getting started
  • Plugins
  • Manual analysis
  • Revision History

What you will need:

  • Santoku Linux Alpha 0.3 (or later)
  • A backup of an iOS device (please refer to libiMobile backup HOWTO)

Getting started:

Using the menu in Santoku, navigate to Santoku –> Device Forensics –> iPhone Backup Analyzer (GUI).

A window will then pop-up asking you to provide the path of your iOS backup:

Note: If you used libimobiledevice backup to create the backup for your iOS device, the path should be similar to: <Path_where_you_saved_your_backups>/<UDID_of_your_device>

Please make sure you double-click on that folder when selecting the backup folder in the iPBA pop-up above.

Once your selection is made, the main window of iPBA will appear:

(Personal information has been removed from screenshots)

The main window in the middle is the description window, which gives you detailed information about a specific selected item. To navigate to a specific item, use the bottom-left pane.

You can either use the pre-configured plugins to access valuable pieces of information from your backup, like call history or SMS, or navigate manually through it to get to the data that interests you. We will give you a few examples of what you can do with iPBA using both methods:

Plugins

To access the plugins, click on the “Plugins” menu at the top of the iPBA window:

If you click on “Call History” for example, you will be presented with a pop-up window that will list all calls present in the backup data:

You can also try the “Contacts Browser” to see the contacts stored in the phone at the time of the backup:

Another interesting artifact to take a look at would be the SMS messages. By clicking on the “SMS Browser” plugin in the menu, you will be presented with a pop-up window, similar to the contact or call history ones, that will allow you to browse the SMS messages stored in the backup:

Finally, another plugin of interest is the “Thumbnails browser” that allows you to navigate through previews of photos stored on the device at the time of the backup:

Manual analysis:

If you want to go deeper in your analysis, you can manually browse for items of interest. This will also allow you to retrieve additional details about the artifacts.

For example, by navigating in the bottom-left panel to “CameraRollDomain –> Media –> Media/DCIM/100Apple”, you will see the listing of media files (pictures and videos) stored in the phone under the Camera Roll folder at the time of the backup.

For example, by clicking on the first .jpg image listed and selecting the “Description” pane, we get alot of useful information about the photo, including potentially valuable data from EXIF tags, such as date and time the photo was taken, GPS location data about where the photo was taken, and more.

iPBA also has a powerful SQLite viewer that allows you to browse data inside those databases directly from the application. For example, if we wanted to see the Safari bookmarks stored in this backup, using the bottom-left pane we would navigate to “Home Domain –> <Library/Safari> –> bookmarks.db.

On the bottom-right pane you will now see the “Tables” pane being populated with the listing of tables contained in this database.

If you click on “bookmarks” you will be able to see the different Safari bookmarks in the middle pane:

While there is still quite a bit to be explored using iPBA, the scope of this tutorial aims at getting you started with using iPBA.
